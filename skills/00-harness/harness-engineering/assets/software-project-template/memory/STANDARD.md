# Memory Standard

## Purpose

Preserve durable project facts that future agents should rely on.

## Save

- Build, test, lint, and run commands.
- Architecture decisions and boundaries.
- Domain vocabulary.
- Project conventions.
- Gotchas and known flaky behavior.
- Important decisions from completed work.

## Do Not Save

- Secrets or credentials.
- Temporary hypotheses.
- Long chat transcripts.
- Facts that are not confirmed.

## Location

Use `memory/project-memory.md` for stable facts and append dated entries only when the fact is likely to matter later.
