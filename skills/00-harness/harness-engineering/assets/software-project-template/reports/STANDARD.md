# Reports Standard

## Purpose

Collect final delivery evidence for coding-manager, reviewers, and users.

## Final Acceptance Report

For each completed feature or fix, create:

```text
reports/YYYY-MM-DD-<short-topic>-acceptance.md
```

Include:

- PRD or issue reference.
- Implementation summary.
- Test report links.
- Debug report links, if any.
- Review report link.
- Acceptance criteria verdict.
- Known residual risks.
- Final status: `ACCEPTED` or `BLOCKED`.
